# 채용알림봇 사용설명서 — 노트북 24시간 운영

봇이 자동으로 채용공고를 카톡 오픈채팅방에 올리려면, 아래 **필수 항목**이 지켜져야 합니다.
하나라도 안 되면 게시가 멈추거나 실패합니다. (API가 아니라 카톡 PC 화면을 자동 조작하는 방식이라
그렇습니다.)

---

## ✅ 필수 (이게 안 되면 작동 안 함)

1. **카카오톡 PC 로그인 유지**
   - 노트북에 카카오톡 PC가 설치·로그인된 상태여야 함.

2. **대상 오픈채팅방을 "창"으로 열어두기**
   - 방을 더블클릭해 **별도 창으로 띄워 두고, 닫지 말 것.**
   - 창 제목이 `config.yaml`의 `kakao_room` 값과 **정확히 같아야** 함
     (현재: `[수습/기합] KICPA(공인회계사) 채용공고 알리미`).
   - 방 이름을 바꾸면 `config.yaml`의 `kakao_room`도 같이 바꿔야 함.

3. **화면 잠금·절전 끄기** ⚠️ 가장 흔한 실패 원인
   - Windows 설정 → 시스템 → 전원 및 배터리 → **화면/절전 = "안 함"**(전원 연결 시).
   - 잠금화면 상태에선 카톡에 입력을 넣을 수 없어 **전송이 전부 실패**함.
   - 화면 보호기도 끄기.

4. **노트북 전원 켜두기** (전원 어댑터 연결 권장).

5. **루프 실행 유지**
   - 게시는 `python -m src.run --loop` 가 떠 있는 동안만 동작.
   - **터미널 창을 닫으면 멈춤.** 항상 떠 있게 하려면 아래 "부팅 시 자동 시작" 설정.

6. **게시 순간 노트북 건드리지 않기**
   - 게시할 때 약 1초간 마우스·창 포커스를 가져감. 그때 다른 창을 클릭하면 엉뚱한 곳에 입력될 수 있음.
   - 가능하면 **전용 노트북**(평소 안 쓰는 기기)으로 돌리는 게 안전.

---

## ▶ 실행 방법

```
python -m src.run --loop
```
- 3시간마다 6개 사이트 검색 → 새 공고만 방에 게시.
- 매일 오전 9시 그날 올린 공고를 요약(다이제스트) 1회 게시.
- 인터넷이 끊겼다 복구되면 **즉시 1회** 검색(자동).
- 노트북을 새로 켜면(자동 시작 설정 시) 시작 직후 1회 검색.

중지: 터미널에서 **Ctrl + C**.

---

## ⚙ 부팅 시 자동 시작 — 현재 해제됨

(GUI 자동 게시가 불안정해 일단 자동시작은 꺼둔 상태)
- **켜기**: `Win+R` → `shell:startup` → 그 폴더에 `start_bot.bat`의 바로가기를 넣으면 로그온 시 자동 실행.
- **수동 실행**: 프로젝트 폴더의 `start_bot.bat` 더블클릭(또는 `python -m src.run --loop`).
- **봇 중지**: 콘솔 창에서 **Ctrl+C** 또는 창 닫기.

> ⚠️ 알림봇(상시 자동 게시)으로는 카톡 GUI 자동화가 불안정해 **현재 권장하지 않음**.
> 대신 아래 "활용 아이디어"의 안정적인 방식(웹 대시보드/텔레그램·디스코드/이메일)을 권장.

---

## 🔧 문제가 생기면

- **방에 안 올라옴**: ① 화면 잠금/절전 꺼졌는지 ② 카톡 방 창이 열려 있는지 ③ 방 제목 = `kakao_room` 인지 확인.
- **엉뚱한 곳에 입력됨**: 게시 순간 노트북을 안 만지면 해결. 전용 기기 권장.
- **카톡 업데이트 후 안 됨**: 입력창 위치가 바뀐 것 → 개발자(클로드)에게 알리면 `kakao_pc.py` 보정.
- **공고 조정**: 분야 키워드·게시 주기·다이제스트 시각은 `config.yaml`에서 변경.

## 💡 활용 아이디어 (스펙쌓기 — 회계×개발)

지금 만든 **6개 소스 스크래퍼 + 어댑터 구조**는 그대로 두고, "게시 방식"만 바꾸면 더 안정적이고
포트폴리오로도 좋은 형태가 된다. 추천 순:

1. **채용공고 웹 대시보드** (가장 추천) — Streamlit/Flask로 6개 사이트 통합 목록 + 필터(분야/마감임박/
   법인별) + 마감 캘린더. 카톡 ToS·GUI 불안정 문제 없음. 풀스택+데이터시각화 어필.
2. **텔레그램/디스코드 봇** — 똑같은 스크래퍼 → 공식 봇 API(무료·합법·안정). "RPA 대신 API를 택한
   판단"을 보여주는 게 오히려 강점. 노트북 없이 클라우드 상주 가능.
3. **이메일 일일 다이제스트** — SMTP로 매일 1통. GUI 불필요, 가장 단순·안정. 빠른 완성 가능.
4. **채용 데이터 분석** — 공고를 DB에 누적 → 법인별·시즌별·분야별 채용 트렌드 분석(pandas/노트북).
   회계사 도메인 지식 + 데이터분석 = 차별화 포인트.
5. **DART 전자공시 스크래퍼로 확장** — 같은 스크래핑 역량을 감사보고서/공시 수집에 적용 →
   [[감사조서 자동화]] 프로젝트와 시너지.

## 🧰 재사용 가능한 도구/자산 (이번에 만든 것)

- **소스별 어댑터 패턴** (`src/adapters/`, `src/record.py`) — 사이트마다 다른 HTML/JSON을 공통 레코드로
  수렴. 새 소스는 어댑터 1개만 추가. (조서 프로젝트의 '이상적 양식 중간다리'와 같은 발상)
- **`src/http_util.py`** — 세션 재사용·재시도·인코딩 보정·커스텀 헤더/JSON 바디. 모든 스크래핑에 재사용.
- **견고성 패턴** (`safe_fetch`, 부분실패 허용) — 한 소스 깨져도 전체 안 죽음.
- **상태/중복제거** (`src/state.py`) — 키 기반 JSON 영속(해시 캐시 발상).
- **사이트 정찰 노하우** (각 어댑터 docstring에 기록):
  - KICPA: `list.face` 정적 + 상세에서 마감일 보강
  - 삼정KPMG: `search.hr` POST → HTML 조각
  - 안진: WiseRecruit2(ASP.NET) 정적 테이블
  - 한영: recruiter.co.kr **JSON API**(`/position/v1/jobflex`, `prefix` 헤더)
  - 삼일PwC: experienced.html Grade 컬럼 + 봇 UA 403 회피
- **GUI 검증 기법** — pywinauto `capture_as_image()`로 창을 스크린샷 떠서 멀티모달 확인(장님 자동화 탈피).
- **서브에이전트 병렬화** — 독립 작업(한영/삼일 어댑터)을 동시 작성 후 문제 교차 공유.

## 📌 참고 (위험·정책)

- 카톡 PC 자동화는 약관상 회색지대 → 최악의 경우 **계정 정지** 가능(형사 불법은 아님). 그래서 저빈도
  (3시간)·소량으로 운영. 실질 정지 위험은 낮은 편.
- 방을 **공지방(방장만 글쓰기)**으로 두면 운영이 더 깔끔.
